What settings actually control

These are not cosmetic preferences. They shape product behavior: when translation fires, how much text is allowed, which languages matter, and whether local testing knobs appear.

isEnabledMaster switch for the whole extension
hoverDelayMsHow long the user must pause before hover work begins
maxSelectionLengthUpper bound that prevents giant requests
learningLanguage / nativeLanguage / bridgeLanguageThe teaching triangle DutchMate is optimized around
providerEndpointUsually hidden from normal users, but useful for local testing

Boundary lesson

Validation happens before saving. That stops weird values from leaking into the rest of the extension and creating harder-to-debug bugs later.

Saving settings is really a safety checklist

This snippet from [src/options/index.ts](/home/mgurram/MGurramAI/projects/dutchmate-proj/dutchmate/src/options/index.ts:144) reads current values, validates them, normalizes language roles, and only then writes to browser storage.

CODE 
async function saveSettings(): Promise<void> {
  const currentSettings = await readSettings();
  const endpoint = providerEndpoint
    ? providerEndpoint.value.trim()
    : currentSettings.providerEndpoint;
  const hoverDelayValue = readNumberInput(hoverDelayMs, defaultSettings.hoverDelayMs);
  const maxSelectionLengthValue = readNumberInput(
    maxSelectionLength,
    defaultSettings.maxSelectionLength,
  );
  const endpointError = validateProviderEndpoint(endpoint);
  const hoverDelayError = validateHoverDelayMs(hoverDelayValue);
  const selectionLengthError = validateMaxSelectionLength(maxSelectionLengthValue);
  const languageRoles = normalizeLanguageRoles(currentLanguageRoles);

  if (endpointError) {
    showStatus(endpointError, "error");
    providerEndpoint?.focus();
    return;
  }
